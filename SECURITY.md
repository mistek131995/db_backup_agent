# Security Policy

## Поддерживаемые версии

Исправления безопасности выходят для последнего минорного релиза и для последнего коммита в `main`. Более старые версии не патчатся — обновляйтесь.

| Версия | Поддержка |
|---|---|
| `1.0.x` | ✅ |
| `< 1.0` | ❌ |

## Куда сообщать

Приватно, одним из способов:

- **Email:** [support@backupster.io](mailto:support@backupster.io) — тема `[security] <краткое описание>`.
- **GitHub Security Advisories:** вкладка *Security → Report a vulnerability* в этом репозитории.

Публичные issue и pull request'ы для уязвимостей не использовать — фикс должен выйти раньше, чем о проблеме узнает атакующий.

В отчёте пригодится: затронутая версия/коммит, шаги воспроизведения, ожидаемое и фактическое поведение, предполагаемое влияние. PoC приветствуется, но не обязателен.

## Сроки ответа (SLA)

| Этап | Срок |
|---|---|
| Подтверждение получения отчёта | 72 часа |
| Первичный triage и оценка severity | 7 дней |
| Фикс critical-уязвимости | 30 дней |
| Фикс прочих — | best effort |
| Публикация advisory | после релиза фикса, координированно с репортёром |

Если в указанные сроки не ответили — продублируйте письмо или отметьте это в advisory.

## Scope

**В scope:**

- Обход или подделка `X-Agent-Token` / эскалация на стороне агента.
- Утечка credentials (БД, S3/SFTP, encryption key) за пределы хоста агента.
- Некорректное применение AES-256-GCM (подмена кадров, переиспользование nonce, обход AAD).
- RCE, path traversal, SSRF в коде агента.
- Повреждение/подмена зашифрованных бэкапов или манифестов.
- Небезопасные значения по умолчанию, которые компрометируют хост агента.

**Вне scope:**

- Атаки, требующие уже имеющегося root/Administrator на хосте агента.
- DoS на собственный хост, где запущен агент.
- Уязвимости в сторонних зависимостях без рабочего вектора через агент (сообщайте мейнтейнерам upstream).
- Социальная инженерия, фишинг сотрудников, физический доступ.
- Отсутствие security-заголовков или rate-limit на дашборде (это отдельный продукт).

## Safe harbor

Добросовестное исследование в рамках этой политики не будет преследоваться. Условия: не уничтожать и не эксфильтровать чужие данные, не нарушать работу чужих инсталляций, дать разумное время на фикс до публикации.
